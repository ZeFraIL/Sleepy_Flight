# Описание класса: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность/Экран)
*   **Назначение:** Это основной экран приложения. Он отвечает за управление пользовательским интерфейсом (UI), управление музыкальным плеером (MediaPlayer) и проверку условий для воспроизведения звука.
*   **Взаимодействие:** Взаимодействует с `AirplaneReceiver` и `HeadphonesReceiver` для получения обновлений о состоянии системы. Также управляет `MediaPlayer` для воспроизведения белого шума.

## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `mediaPlayer` | `MediaPlayer` | Воспроизводит аудиофайл (белый шум). | В `onCreate`, `checkConditionsAndPlay` и `onDestroy`. |
| `isAirplaneOn` | `boolean` | Хранит состояние режима полета (вкл/выкл). | Обновляется в `AirplaneReceiver`, проверяется в `checkConditionsAndPlay`. |
| `isHeadphonesOn` | `boolean` | Хранит состояние наушников (подключены/нет). | Обновляется в `HeadphonesReceiver`, проверяется в `checkConditionsAndPlay`. |
| `playButton` | `Button` | Кнопка, на которую нажимает пользователь для запуска/остановки звука. | В `onCreate` для установки слушателя нажатий. |
| `airplaneStatus` | `TextView` | Отображает текстовый статус режима полета. | В `updateUI`. |
| `headphonesStatus` | `TextView` | Отображает текстовый статус наушников. | В `updateUI`. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Содержит сохраненные данные предыдущей сессии (если есть). |
*   **Что делает метод:**
    1.  Устанавливает визуальную разметку через `setContentView`.
    2.  Находит и связывает элементы UI (кнопку, текстовые поля) через `findViewById`.
    3.  Инициализирует `MediaPlayer` конкретным звуковым ресурсом (`R.raw.white_noise`).
    4.  Устанавливает зацикливание музыки.
    5.  Устанавливает слушатель "onClick" для кнопки воспроизведения.
    6.  Регистрирует "Ресиверы" для прослушивания системных изменений (режим полета и подключение гарнитуры).
    7.  Вызывает `updateUI` для отображения начального состояния.
*   **Когда вызывается:** Автоматически системой Android при запуске приложения.
*   **Что важно понимать:** Это точка входа. Если забыть инициализировать что-то здесь, приложение может аварийно завершиться позже.

### Имя метода: `checkConditionsAndPlay`
*   **Тип:** `public static`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает метод:**
    1.  Проверяет, является ли `isAirplaneOn` или `isHeadphonesOn` ложным (false).
    2.  Если условия НЕ соблюдены: останавливает музыку, если она играет.
    3.  Если условия СОБЛЮДЕНЫ: переключает музыку (запускает, если остановлена, и наоборот).
    4.  Вызывает `updateUI` для обновления текста на экране.
*   **Когда вызывается:** При нажатии кнопки или когда Ресивер обнаруживает изменение режима полета или наушников.
*   **Что важно понимать:** Метод является `static`, что позволяет Ресиверам вызывать его напрямую. Однако текущая логика работает как переключатель при соблюдении условий.

### Имя метода: `updateUI`
*   **Тип:** `public static`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает метод:** Обновляет текст на экране, показывая "Активен/Выкл" для режима полета и "Подключены/Отключены" для наушников.
*   **Когда вызывается:** При каждом изменении состояния.

### Имя метода: `onDestroy`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает метод:** Освобождает ресурсы `MediaPlayer` обратно системе.
*   **Когда вызывается:** При закрытии или уничтожении приложения.
*   **Что важно понимать:** Критически важно для предотвращения утечек памяти. Если не "освободить" плеер, он может продолжать потреблять батарею/память даже после закрытия приложения.

## 4. Жизненный цикл
*   **onCreate():** Вызывается при создании Activity. Здесь мы настраиваем «сцену» нашего приложения.
*   **onDestroy():** Вызывается перед уничтожением Activity. Мы используем его для очистки ресурсов.

## 5. Взаимодействие с интерфейсом (UI)
*   **Кнопки:** `playButton` — обрабатывает нажатия для переключения звука.
*   **Текстовые поля:** `airplaneStatus`, `headphonesStatus` — обеспечивают визуальную обратную связь.
*   **findViewById:** Используется для связи XML-разметки с Java-переменными.

## 6. Взаимодействие с другими компонентами
*   **Ресиверы:** Класс использует `registerReceiver` для прослушивания системных сообщений.
*   **MediaPlayer:** Системный компонент для обработки воспроизведения аудио.

## 7. Общая логика класса
Класс выступает в роли координатора. Он ждет, когда пользователь захочет включить музыку, но работает как «охранник», который разрешает музыку только если телефон в авиарежиме и наушники подключены.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте `MainActivity` как **Капитана самолета**. У капитана есть контрольный список перед взлетом (воспроизведением музыки).
- Один пункт — «Режим полета».
- Другой — «Наушники подключены».
Капитан не запустит двигатели (MediaPlayer), пока не будут отмечены оба пункта. Если вы летите и вдруг выдергиваете наушники, Капитан немедленно глушит моторы, чтобы вы не беспокоили других пассажиров.

---
**Примечание по улучшению:** Использование `public static` для элементов UI может привести к утечкам памяти при повороте экрана. Лучше использовать ViewModel.
