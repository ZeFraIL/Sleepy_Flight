# Описание класса: AirplaneReceiver

## 1. Общая информация
*   **Имя класса:** `AirplaneReceiver`
*   **Тип:** BroadcastReceiver (Приемник широковещательных сообщений)
*   **Назначение:** Этот класс «слушает» изменения статуса режима полета в устройстве. Он работает как автоматический наблюдатель, который «просыпается», когда система посылает сигнал.
*   **Взаимодействие:** Сообщает `MainActivity` об изменениях, обновляя её переменные и запуская логику проверки.

## 2. Переменные (Поля класса)
У этого класса нет собственных постоянных переменных. Он использует локальные переменные внутри методов.

## 3. Методы класса

### Имя метода: `onReceive`
*   **Тип:** `public`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `context` | `Context` | Окружение приложения. |
    | `intent` | `Intent` | Сообщение от системы с деталями события. |
*   **Что делает метод:**
    1.  Проверяет, действительно ли полученное сообщение касается изменения режима полета.
    2.  Извлекает состояние (True — включен, False — выключен).
    3.  Обновляет переменную `MainActivity.isAirplaneOn`.
    4.  Показывает "Toast" (короткое всплывающее уведомление) для пользователя.
    5.  Вызывает `MainActivity.checkConditionsAndPlay()` для немедленной реакции.
*   **Когда вызывается:** Автоматически системой Android, когда пользователь переключает режим полета в настройках.
*   **Что важно понимать:** `BroadcastReceiver` — это временные объекты. Они существуют только во время выполнения метода `onReceive`. В них нельзя выполнять долгие задачи.

## 4. Жизненный цикл
Этот класс не является Activity, поэтому у него нет стандартного жизненного цикла. Только событие `onReceive`.

## 5. Взаимодействие с интерфейсом (UI)
*   **Toast:** Использует `Toast.makeText` для показа краткого сообщения. Постоянного интерфейса нет.

## 6. Взаимодействие с другими компонентами
*   **MainActivity:** Напрямую меняет статические поля.
*   **System Intent:** Слушает `Intent.ACTION_AIRPLANE_MODE_CHANGED`.

## 7. Общая логика класса
Этот класс похож на **Датчик**. Он ничего не делает, пока не почувствует изменения. Как только это происходит, он сообщает в «главный офис» (`MainActivity`), чтобы тот решил, что делать дальше.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте `AirplaneReceiver` как **Охранника**, стоящего у переключателя режима полета. 
Охранник не управляет самолетом, но как только он видит, что кто-то щелкнул тумблером, он берет рацию и кричит Капитану (`MainActivity`): «Сэр, режим изменился! Проверьте, можно ли нам еще играть музыку!»
Также охранник показывает пользователю маленькую табличку (Toast), чтобы тот знал, что изменение замечено.
